Custom implementations

During the development of the project, and as the only team of 2017 implementing our system in Python, we found several function implementations that were easily done in MATLAB but did not have a good Python counterpart. Some of these alternatives presented additional issues that hintered our progress. Some of these issues were difficult installations, the lack of a multiplatform implementation, lack of easy to use documentation or simply absence. We made the best effort to try to overcome these obstacles, and for the use of the next master students, we have created a small list with all the functionalities OpenCV we could not find in OpenCV or standard Python libraries:

  • Kalman Filter : we implemented our own.
  • MATLAB function bwareaopen : we implented our own.
  • Hole filling : we implemented our own with the help of scipy-image library.
  • Lucas Kanade : although OpenCV offers a sparse LK version, it lacks a dense counterpart. Moreover, there are no more Dense Optical Flow implementations in OpenCV, apart from Farneback.
Despite these lacks, we enjoyed developing our code in Python, as we believ is a more robust and accesible language, aimed at a wider range of tasks than MATLAB. The access to some of its powerful libraries made some of the very needed non computer vision tasks such managing files or communicating with the Operative System much easier. Needless to say, we are open to point at the specific code locations where these problems were solved.
